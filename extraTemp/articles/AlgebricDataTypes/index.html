What are Algebraic types

Algebraic data types is a concept important for Functional Programming.

Sum Types

In Swift, the Enum is a sum type.

A sum type has, in total, the sum of possible values of each type.

A sum type is usefull to express "disjunction". Something as "this value can be an A or a B".

Product Types

In Swift, a Struct is a product type.

A product type has, in total, the product of possible values of each type.

A product type is usefull to express "conjunction". Something as ""this value can be an A and a B"."

What's the purpose

Most of the times, as a developer, we only use 10 or 20% of the capabilities of a language. We define some data types, built it other data types or language's primitive types, make use of collections and the flow control mechanisms to control the data. At the end, some of this data end up being formated and displayed on screen inside a rectangle.

However, languages are much more powerful tools. Undoubtly that knowing how to use those advanced tools is positive. The point is, do I need to know those advanced tools in my day-to-day tasks? Where can I use them correctly?

In a more functional approach, the knowledge and use of algebraic data types make more sense.

Stretch Goal

One of the problems functional programming and algebraic types might help solving is the expression problem.

Expression Problem

The expression problem is a new name for an old problem.2 The goal is to define a datatype by cases, where one can add new cases to the datatype and new functions over the datatype, without recompiling existing code, and while retaining static type safety (e.g., no casts).
