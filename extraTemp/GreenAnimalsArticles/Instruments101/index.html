Custom Instruments

Based on WWDC19 session: Creating Custom Instruments (410)

Architecture

instruments has two main components:

  • StandardUI
  • Analysis Core

StandardUI is tightly coupled with Analysis Core. Analysis Core is kind of a Database and a expert system.

StandardUI

  • Track View
    • Multiple Graphs
      • Multiple lanes
  • Detail View
    • Event Views

Diference between a plot and a plot-template schema is that the template is dynamically generated. The number of lanes in a graph is not static, is defined by data.

The Details can be of multipe types:

  • List
  • Aggregations
  • Call Tree
  • Narrative
  • Time Slice

Analysis Core

Data is stored in tables

A table has a schema.

Binding solutions. An optimization that will allow multiple instruments operating similar (or the same operation) to use the same data and operations in order to optimize resouces.

There is more than 100 schemas available. We can use those schemas. Eventually we need to Link (build settings) those "subschemas" in order to import the schema.

Modelers

Modelers are kind of Expert Systems and are written in CLIPS language.

In order to help us with Modelers (the high level thing), we have Automatic Modelers. Have a look on <os-signpost-interval-schema>.
